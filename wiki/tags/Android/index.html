相关编译错误

Error running app：Default Activity not found

在任务清单内为活动注册事件

提示Installation failed

  1. 检查包名是否存在大写字母,改为小写字母即可

  2. 检查setting下的build内install run第一选择是否被选中,若此则取消选中,然后保存即可

Unable to find optional library: org.apache.http.legacy

HttpClient已经在API 23中删除
在相应的module下的build.gradle中加入即可

org.gradle.api.GradleException: Lint found fatal errors while assembling a release target

  1. 日志详情
  2. 原因分析
    Lint在编译release包的时候发现了致命的错误。
  3. 解决方法
    方法一：
    为了继续编译或解决Lint标记的问题，或许应该对你的构建脚本进行如下的改动，即在app主module的build.gradle里增加lintOptions，其中checkReleaseBuilds false表示在进行Release构建时不再进行Lint检查，abortOnError false则表示检查到错误后继续编译，不取消当前的构建任务。

方法二：
通过release编译打包时的编译打印日志内容中错误提示信息跳转对应布局文件中解决问题

Error message “Android Gradle plugin requires Java 11 to run. You are currently using Java 1.8”

Gradle JDK选择11 version 11.0.8/opt/android-studio-canaryl/jre即可

Caused by: java.lang.IllegalArgumentException: The given artifact contains a string literal with a package reference ‘android.support.v4.content’ that cannot be safely rewritten

  1. 原因
    support.v4库未找到Androidx支持类型

相关运行异常

net::ERR_CLEARTEXT_NOT_PERMITTED

  1. 原因
    Android9.0以上无法直接加载http
  2. 多种解决方法
    一. application节点添加
    二. targetSdkVersion 降级回至27
    三. 创建xml新建network_security_config
    在清单文件的application标签添加配置
    四. 服务器和本地应用都改用 https

The style on this component requires your app theme to be Theme.AppCompat (or a descendant).

在引用这个自定义布局的界面根标签里，添加一行代码

Failure delivering result ResultInfo{. }to activity java.lang.IllegalStateException: Can not perform this action after onSaveInstanceState

  1. 原因
    activity不可见时不执行数据文件传值操作
  2. 解决方法
    通过趣味问题的第一点解决 让这种逻辑处于activity可见时执行

Parameter specified as non-null is null: method kotlin.jvm.internal.Intrinsics.checkParameterIsNotNull, parameter favicon

  1. 原因
    webview监听favicon kotlin语法中需要判断为空可否
  2. 解决方法
    重写的onPageStarted方法类的favicon: Bitmap Bitmap后添加?使其可空即可

Recycleview data is not displayed

  1. 原因
    多重xml布局 recycleview id相同
  2. 解决方法
    修改ID为其他名称
